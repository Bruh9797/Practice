# Авторизация и доступ

ThermoSelect использует серверную `HttpSession`. При JSON-входе идентификатор сессии изменяется, cookie `JSESSIONID` имеет `HttpOnly` и `SameSite=Lax`, время бездействия — 30 минут.

## CSRF

1. Клиент выполняет `GET /api/auth/csrf`.
2. Полученный токен хранится только в памяти вкладки.
3. Во все изменяющие запросы передаётся заголовок, указанный в ответе (`X-XSRF-TOKEN`).
4. После входа и выхода клиент получает новый токен.

## Роли

- `USER` — каталог, поиск, карточка и сравнение;
- `ADMIN` — возможности USER, редактирование каталога и управление пользователями.

Состояние и роль пользователя перечитываются из БД на каждом защищённом запросе. Блокировка, удаление и смена роли поэтому действуют сразу для уже открытой сессии.

Полная матрица приведена в [`api-access.md`](api-access.md).
