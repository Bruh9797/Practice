# Алгоритм фильтрации и ранжирования

## Назначение

Алгоритм выбирает записи каталога, которые подтверждённо удовлетворяют введённым ограничениям, а затем упорядочивает их по качеству совпадения. Он не выполняет тепловой расчёт и не заменяет проверку производителя.

## Этапы

1. Нормализовать строку поиска и коды справочников; проверить `page >= 0`, `1 <= size <= 100` и неотрицательные числовые требования.
2. Получить только записи `PUBLISHED`.
3. Применить строгие категории: семейство, производитель, все выбранные области применения и материалы.
4. Для текстового запроса проверить производителя, модель, серию, описание, применения, материалы и специальные факты.
5. Для каждого заданного числового критерия:
   - если паспортного значения нет, исключить запись и увеличить `excludedUnknownCount`;
   - если ограничение не выполняется, исключить запись;
   - иначе вычислить близость и сформировать понятную причину совпадения.
6. Рассчитать внутренний коэффициент порядка и полноту паспортных данных, затем отсортировать результаты.
7. Применить стабильную пагинацию и вернуть результаты и число исключённых записей с неизвестными обязательными значениями. Внутренний коэффициент не показывается пользователю как процент совпадения.

## Весовые группы

| Группа | Базовый вес | Когда активна |
| --- | ---: | --- |
| Текстовое совпадение | 30 | введён непустой текст |
| Области применения | 20 | выбран хотя бы один код |
| Материалы | 10 | выбран хотя бы один код |
| Числовая близость | 30 | задан хотя бы один числовой критерий |
| Гранулярность данных | 10 | всегда |

Активные веса нормализуются:

```text
score = round(100 × Σ(weightᵢ × groupScoreᵢ) / Σ(weightᵢ))
```

Коэффициент гранулярности: точная конфигурация — `1.00`, стандартная модель — `0.80`, серия — `0.55`.

### Числовая близость

Для минимально необходимой способности (`площадь`, `максимальное давление`):

```text
closeness = required / available
```

Подходящая модель с меньшим избыточным запасом получает более высокий балл.

Для попадания в опубликованный диапазон (`расход`, `температура`) оценивается расстояние до середины диапазона; балл ограничен интервалом `0.5…1.0`.

## Разрешение равенств

1. внутренний коэффициент по убыванию;
2. полнота известных паспортных полей по убыванию;
3. производитель по алфавиту;
4. модель по алфавиту;
5. числовой ID.

## Пояснения API

API может сформировать массив `reasons` для диагностики алгоритма, однако интерфейс не перегружает карточки служебными процентами и пояснениями. Примеры:

- «Максимальное давление покрывает требование: 30 bar»;
- «Требуемый расход входит в диапазон 10–100 м³/ч»;
- «Параметры относятся к точной конфигурации»;
- «Данные относятся к серии — конфигурацию необходимо уточнить у производителя».
