# Backend и каталог

Backend находится в `Авторизация/src/main/` и использует Spring Boot 4.1, Spring Data JPA, Flyway и Hibernate в режиме `ddl-auto=validate`.

## Состав

- `catalog/` — сущности, DTO, поиск, ранжирование, сравнение и ADMIN CRUD;
- `security/` — пользователи, роли, session-аутентификация и CSRF;
- `web/` — точный SPA fallback;
- `resources/db/migration/` — схема БД;
- `resources/data/demo-catalog.psv` — 50 записей каталога.

## Данные каталога

Загрузчик создаёт отсутствующие позиции и дозаполняет пустые характеристики в существующей H2/PostgreSQL базе. Непустые значения, которые администратор уже скорректировал вручную, не перезаписываются.

Для всех семейств применяется единый набор общих характеристик: площадь поверхности, минимальный и максимальный расход, минимальная и максимальная температура, максимальное рабочее давление, ширина, высота, глубина и масса. Мощность исключена из поиска и публичной карточки, поскольку определяется рабочим режимом, а не только моделью аппарата. Типоспецифичные параметры хранятся отдельными фактами.

Подтверждённые значения сохранены из официальных источников. Если отдельная характеристика отсутствует в открытом материале, в набор данных внесено согласованное справочное значение для соответствующего типоразмера. Политика источников описана в [`catalog-sources.md`](catalog-sources.md).

`POST /api/heat-exchangers/export` принимает тот же фильтр, что и поиск, и формирует Excel-книгу с автофильтром, закреплённой строкой заголовка и едиными числовыми форматами.

## Запуск API

```powershell
cd .\Авторизация
.\mvnw.cmd spring-boot:run "-Dspring-boot.run.profiles=demo"
```

Для рабочего запуска рекомендуется единый JAR через `run-demo.ps1`, описанный в корневом README.
