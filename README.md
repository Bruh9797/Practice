# Учебная практика: информационно-поисковая система для выбора теплообменных аппаратов

## Общая информация

Тема практики: разработка информационно-поисковой системы для выбора теплообменных аппаратов.

Период практики: 06.07.2026-19.07.2026.

Срок сдачи отчета: 19.07.2026.

Цель работы: спроектировать и разработать информационную систему, которая помогает пользователю находить и выбирать теплообменные аппараты по заданным параметрам, а администратору - управлять данными, доступом, журналами и отчетностью.

## Распределение обязанностей

### Артем - фронтенд поисковой системы

Артем отвечает за пользовательскую часть информационно-поисковой системы.

Основные задачи:

- разработка интерфейса поиска теплообменных аппаратов;
- создание формы ввода параметров и фильтров;
- отображение результатов поиска в виде карточек или таблицы;
- подготовка экранов для пользователя и администратора, если они нужны для демонстрации;
- интеграция интерфейса с API бэкенда;
- обработка состояний загрузки, ошибок и отсутствия результатов;
- участие в подготовке скриншотов и описания пользовательского интерфейса для отчета.

Ожидаемые результаты:

- рабочий интерфейс поисковой системы;
- понятная навигация по основным функциям;
- демонстрация поиска и фильтрации теплообменных аппаратов.

### Сережа - бэкенд и база данных

Сережа отвечает за серверную часть системы и хранение данных.

Основные задачи:

- проектирование структуры базы данных теплообменных аппаратов;
- описание основных сущностей, параметров и связей;
- реализация хранения данных о теплообменниках;
- разработка API для поиска, фильтрации и получения карточек аппаратов;
- обработка запросов от фронтенда;
- подготовка тестовых данных для демонстрации;
- участие в описании концептуальной и даталогической моделей базы данных для отчета.

Ожидаемые результаты:

- структура базы данных;
- серверная логика поиска;
- API, через которое фронтенд получает данные.

### Тигран - авторизация и права доступа

Тигран отвечает за систему входа пользователей и разграничение прав.

Основные задачи:

- проектирование ролей пользователя и администратора;
- реализация входа в систему;
- настройка проверки прав доступа к функциям;
- ограничение административных действий для обычных пользователей;
- взаимодействие с бэкендом и фронтендом для подключения авторизации;
- описание сценариев пользователя и администратора для UML-диаграмм;
- участие в подготовке раздела отчета по безопасности и правам доступа.

Ожидаемые результаты:

- система авторизации;
- роли и права доступа;
- защищенные административные функции.

### Милан - журналы, отчеты и тестирование

Милан отвечает за контроль работы системы, отчетные функции и проверку качества.

Основные задачи:

- разработка системы журналирования действий пользователей и ошибок;
- подготовка отчетных форм или выгрузок по результатам работы системы;
- составление тест-кейсов для основных сценариев;
- проверка поиска, авторизации, прав доступа и обработки ошибок;
- фиксация найденных проблем и результатов тестирования;
- участие в оформлении итогового отчета и презентации;
- подготовка раздела о тестировании системы.

Ожидаемые результаты:

- журналы действий и ошибок;
- отчетные материалы;
- список тест-кейсов и результаты проверки системы.

## Совместные задачи команды

Все участники совместно выполняют следующие работы:

- согласование структуры системы и границ ответственности модулей;
- анализ предметной области и подбор источников;
- подготовка UML-диаграмм и описания архитектуры;
- интеграция фронтенда, бэкенда, базы данных, авторизации и отчетности;
- подготовка итогового отчета по практике;
- подготовка презентации для защиты;
- финальная проверка демонстрационного сценария.

- тест-кейсы и результаты тестирования;
- итоговый отчет;
- презентация для защиты.
