# ThermoSelect

Информационно-поисковая система для выбора теплообменных аппаратов. Проект разработан в рамках учебной практики СПбГТИ(ТУ) по направлению 09.03.01 «Информатика и вычислительная техника».

ThermoSelect объединяет React-интерфейс и Spring Boot API в одном исполняемом JAR. Пользователь задаёт значимые паспортные ограничения, получает упорядоченный список, выгружает его в Excel и сравнивает до четырёх аппаратов. Администратор управляет каталогом и правами пользователей.

## Возможности

- регистрация, session-аутентификация и роли `USER` / `ADMIN`;
- защищённый каталог четырёх семейств теплообменников;
- единый набор значимых характеристик и строгие фильтры по ним;
- карточки со ссылками на официальные источники;
- сравнение до четырёх моделей;
- выгрузка отфильтрованного каталога в формат Excel (`.xlsx`);
- CRUD каталога, публикация и мягкое архивирование;
- управление ролями, блокировка и удаление пользователей;
- единая сборка React + Spring Boot, H2 demo и PostgreSQL-профиль.

## Быстрый запуск

Требуется PowerShell и доступ в интернет при первой сборке. Maven и Node загружаются wrapper-плагинами в каталог `target`.

```powershell
cd .\Авторизация
.\run-demo.ps1
```

После успешной сборки приложение доступно по адресу <http://localhost:8080>.

| Роль          | Логин   | Пароль       |
| ------------- | ------- | ------------ |
| Пользователь  | `demo`  | `demo12345`  |
| Администратор | `admin` | `admin12345` |

Профиль `demo` использует файловую H2 и сохраняет изменения между запусками в `Авторизация/data/`. Демо-пароли предназначены только для локального запуска.

## Проверка

```powershell
cd .\Авторизация
.\mvnw.cmd clean verify
```

Команда устанавливает закреплённый Node, выполняет `npm ci`, frontend- и Java-тесты, production-сборку Vite и упаковку единого JAR.

## PostgreSQL

```powershell
$env:DB_URL='jdbc:postgresql://localhost:5432/heat_exchanger_db'
$env:DB_USERNAME='postgres'
$env:DB_PASSWORD='secret'
$env:ADMIN_USERNAME='admin'
$env:ADMIN_PASSWORD='change-this-password'
$env:ADMIN_EMAIL='admin@example.org'
java -jar .\Авторизация\target\heat-exchanger-selector-0.0.1-SNAPSHOT.jar --spring.profiles.active=postgres
```

Для HTTPS установите `SESSION_COOKIE_SECURE=true`. Обычный запуск JAR без явного профиля намеренно не создаёт demo-аккаунты.

## Структура

```text
Авторизация/
├── frontend/                 React 19 + JavaScript + Vite
├── src/main/java/            Spring Boot API, безопасность и каталог
├── src/main/resources/       профили, Flyway и 50 записей каталога
└── src/test/                 Java-тесты
docs/                         модели, UML, алгоритм и тест-кейсы
tools/                        воспроизводимое обогащение каталога и генерация отчёта
Материалы/                    исходные материалы практики (не изменяются)
```

Каталог содержит 50 записей Alfa Laval, Danfoss, Kelvion, SWEP, API Heat Transfer, российского производителя «Ридан» и аппараты советского типоряда ЧЗТО. У каждой записи указана гранулярность: точная конфигурация, стандартная модель или конфигурируемая серия. Общие характеристики приведены к единому набору: площадь поверхности, диапазон расхода и температуры, максимальное давление, габариты и масса. Если открытый источник не содержит отдельного значения, используется согласованное справочное значение, подходящее для соответствующего типоразмера. При запуске приложение также дозаполняет пропуски в ранее созданной файловой базе, сохраняя уже отредактированные значения.

## Ограничения

ThermoSelect — информационно-поисковая, а не расчётная система. Она не выполняет LMTD/sizing: мощность и реальный тепловой режим зависят от сред, расходов, температур и допустимого перепада давления, поэтому мощность не используется как универсальная каталожная характеристика. Цены, наличие, история поиска и изменение пароля в эту версию не входят.
